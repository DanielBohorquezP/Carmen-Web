# TASKS — Well Brain

Documento de tareas del sitio. La guía de marca, contenido y sistema visual vive en `CLAUDE.md`.

## Protocolo de actualización

1. Se trabaja **una tarea a la vez**, en el orden de este documento.
2. **Antes de empezar cualquier tarea, se debe preguntar al usuario si está de acuerdo con el enfoque propuesto y esperar su confirmación explícita.** No se escribe ni edita código sin esa confirmación.
3. **Para cualquier cambio se deben invocar siempre las skills `ui-ux-pro-max:design` y `ui-ux-pro-max:ui-styling` antes de escribir código.**
4. Al terminar una tarea: se marca `[x]`, se actualiza la sección **Siguiente tarea** y se continúa con la próxima solo después de repetir el paso 2.
5. Si una tarea revela trabajo nuevo, se agrega como sub-tarea en su fase correspondiente (no se improvisa fuera del documento).
6. **El servidor de preview (`npm run dev`) se deja corriendo entre tareas y correcciones — no se apaga al terminar una tarea.** Solo se detiene si el usuario lo pide explícitamente.
